# Pixel Angler The Real — Pixel-Perfect Korean Sea Fishing Simulator

> **2D 픽셀 퍼펙트 한국 해양 낚시 시뮬레이터 + 생활 경영 RPG**
> PC (Tauri v2 데스크톱 앱) · 추후 Steam 출시 목표

실제 지형 지도, 실측 연안 수심, 물때(1~15물), 공공 OpenAPI(기상청·해양기상·바다낚시지수·경락 시세·어획량 통계)를
게임 물리에 직접 연동하는 **리얼리즘 낚시 시뮬레이터**입니다.

**1차 테스트 빌드 (GitHub Pages)**: https://sarsah93.github.io/Pixel-Angler-The-Real/

## 핵심 시스템 (구현 현황)

### 월드 & 필드
| 시스템 | 상태 | 설명 |
|---|---|---|
| 전국 지도 → 지역 줌인 | ✅ | 픽셀 전국 지도에서 지역 핀 클릭 → 이음새 없는 확대 전환. 미개방 지역 잠금 |
| 출조 구역 선택 | ✅ | 구역 핀 + 특성 카드(수심/밑걸림 위험/상세) → 출조 확인 → 필드 진입 |
| 실지형 타일맵 필드 | ✅ | 실제 지형을 색상 분류한 탑다운 타일맵 — **속초 7맵** + **부산 8맵**(감천항 서·동방파제/암남/백운포) 엣지 전환 |
| 건물 & 상점 | ✅ | 편의점/식자재마트/직판장/음식점/카페/주점 — 구매·판매, 수량/확인 팝업, 실시세 연동 |
| HUD | ✅ | HP·피로도·시계·날씨(실데이터) / 실지형 미니맵(M 3단계) / 퀵슬롯 8칸 / 이벤트 로그·채팅(멀티 대비) |

### 낚시 파이프라인 (1인칭)
```
탑다운 조준 캐스팅 ─→ 착수 ─→ 1인칭 낚시 뷰 ─→ 입질 시퀀스 ─→ 챔질 ─→ 파이팅 ─→ 랜딩/실패
  마우스 각도+파워      실측 수심   침강·조류 존·정렬도   초릿대 3단계     우클릭    텐션 0~100
  3D 탄도(바람/중력)    여 밭 판정  뒷줄견제(H)·밑밥(C)   구부러짐 읽기            어종별 저항 패턴
```
| 모듈 (`@tra/core`) | 역할 |
|---|---|
| `CastingPhysicsEngine` | 완력×파워×조준 3D 탄도, 바람 편향, 그림자/찌 이원화 |
| `UnderwaterSinkPhysics` | 침강 V=(W−B)/(C×(1+k·유속)), 면사매듭/바닥 안착 |
| `TidalCurrentEngine` | 조수·반탄류·**조경지대**(입질 1.6배 Hit Zone)·횡조류·본류 — 존별 유불리 |
| `LineTensionPhysics` | 뒷줄견제 제동·양력, 목줄 정렬도 A(0~1) |
| `ChumPhysics` | 밑밥 투척·확산·3차원 동조율 |
| `BiteSequenceEngine` | 입질 이벤트 → **초릿대 구부러짐 3단계**(30°/45°/60°) → 챔질 성공률 5%/20%/100% |
| `BiteProbabilityEngine` | P = P_base × M_지형(여밭 Hold 2.5) × (1+k·A) × M_리액션 × M_밑밥 + 밑걸림(지역 위험도 연동) |
| `FishSpawningOracle` | **어종 마스터 43종** — 서식지/수심층/미끼 선호/성전환/금어기·금지체장/물때 활성도 |
| `FightingPhase` | 텐션 상태 머신 — 바늘털이/여박기/횡이동(쓸림) 어종별 패턴, 입 연약도, 목줄 절단 |

- **조작**: 우클릭 = 챔질 · 좌클릭 홀드 = 릴링(화면 좌/우측 방향 당김) · H = 뒷줄견제 · ↑ = 리프트 · 좌클릭 탭 = 호핑 · 더블탭 = 트위칭 · C = 밑밥
- **낚싯대**: 5절 로드 + 스피닝릴 렌더, 초릿대는 항상 물 쪽으로 벤딩. 설정에서 **로드 위치(좌/우)·릴 핸들(좌/우)** 선택
- **조법**: 반유동 / **전유동**(무한 침강+견제 정지) / **잠길찌** / **원투 편대**(T자 천평·카드 채비 열기 7단/고등어 5단/전갱이 3단 — 다관점 히트)
- 낚시 실패 시 유형별 **채비 손실**(미끼 털림/목줄 터짐/찌 터짐/밑걸림) → 재장착 후 캐스팅
- 첫 진입 시 **온보딩 가이드 4페이지**(입질 읽기→챔질 타이밍→채비 다루기→파이팅), F1/? 버튼 재열람

### 인벤토리 & 채비
- 카테고리 탭 × 5x5 소켓, 드래그 앤 드랍, 우클릭 액션(상세보기/좌·우손 착용/채비하기/퀵슬롯)
- 채비 조립 8소켓: 원줄→면사매듭→찌→도래→목줄→봉돌→**바늘(루어)**→**미끼** — 루어 장착 시 미끼 불필요
- 활용(U) 창: 채비하기 + 요리하기(도마+임베드 인벤토리 — 손질 시스템 예정)
- 어획물은 개체별 실측치(어종/길이/무게) 보존 → 어종·등급·길이 기반 정식 판매가 산정

### 실데이터 연동
| 데이터 | 원천 | 게임 반영 | 상태 |
|---|---|---|---|
| 물때 (1~15물) | 음력 계산 | 조류 세기, 어종 활성도 | ✅ |
| 연안 수심 | 국립해양조사원 연안정보도 SHP | 캐스팅 거리→실측 수심 (속초 앵커별) | ✅ |
| 기상청 단기예보 | KMA OpenAPI | HUD 날씨 + **탑다운/1인칭 실시간 하늘·비·눈·안개 렌더** — 라이브 배포에서도 실데이터 | ✅ |
| 해양기상 76개소 | 국립해양측위정보원 | 풍향·풍속·수온(11개소)·시정 | ✅ (dev 프록시) |
| 바다낚시지수 | 국립해양조사원 | 입질 확률 P_base ×0.7~1.4 | ✅ |
| 경락 시세 | 농정원 MAFRA (2023 동월동일) | 직판장 매입가 실시세 | ✅ (dev 프록시) |
| 어획량 통계 | KOSIS 시도별 어종별 | 지역별 어종 스폰 가중 | ✅ (dev 프록시) |

> 전 API는 스타트업 1회 수집 → 캐시 참조 구조. 실패/키 미설정 시 Mock 폴백으로 오프라인 정상 구동.
> 메인 메뉴 좌하단 **"실시간 데이터 연동" 패널**에서 소스별 실데이터/Mock 상태를 확인할 수 있다.
> 하단 바는 서비스 지역별 **정보 티커**로 순환: 실황 환경 + 7단계 낚시 등급 → 경락 시세 TOP5 → 어획량 상위 어종 → 선호 어종 입질 전망.
> 정적 배포(GitHub Pages)에서는 CORS 제약으로 기상청만 라이브 실데이터 — 나머지는 서버 프록시 도입 예정.

## 시작하기

```bash
# 요구사항: Node ≥ 20, pnpm ≥ 9
npx pnpm install

# 개발 서버 (Phaser 게임 — dev에서는 vite 프록시로 전 API 실데이터)
npx pnpm --filter @tra/client-pc run dev   # → http://localhost:5173

# 전체 빌드 / 타입 체크
npx pnpm run build
npx pnpm --filter @tra/client-pc run typecheck
```

### 데이터 파이프라인 재생성
```bash
py tools/build_region_maps.py sokcho    # 지형 지도 → 타일맵 JSON (busan 등 지역명)
py tools/build_depth_profiles.py        # 수심 SHP(09.수심.zip) → 수심 프로필 JSON
```

### API 키 (`packages/client-pc/.env` — 없으면 Mock으로 동작)
| 변수 | 발급처 | 용도 |
|---|---|---|
| `VITE_DATA_GO_KR_API_KEY` | data.go.kr | 바다낚시지수 · 기상청 단기예보 |
| `VITE_MAFRA_API_KEY` | data.mafra.go.kr | 수산물 경락 시세 |
| `VITE_KOSIS_API_KEY` | kosis.kr | 어획량 통계 |
| `VITE_NMPNT_API_KEY` | marineweather.nmpnt.go.kr | 해양기상 76개소 |

## 씬 아키텍처

```
Boot → MainMenu (저장 슬롯) → WorldMap (전국 → 지역 줌인 → 구역 카드)
                                  ↓ scene.start
                             RegionFieldScene (실지형 타일맵 허브)
                                  ↓ pause + launch          ↺ 맵 간 엣지 전환 (restart)
                             FirstPersonFishingScene (1인칭 낚시)
                                  └ stop + resume → 필드 복귀 (위치 보존)
```

> 규칙: 하위 씬에서 `scene.start(허브씬)` 금지 — 반드시 `scene.stop()` + `scene.resume()` (상세: `.agents/AGENTS.md`)
